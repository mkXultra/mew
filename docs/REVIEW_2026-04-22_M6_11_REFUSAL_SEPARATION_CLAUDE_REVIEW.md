## 2. Does the dogfood now prove the observer bridge?

**Yes.** With the strip in place, the chain that the dogfood actually exercises is:

1. State on disk has a session with `model_turns[0]` carrying `model_metrics.tiny_write_ready_draft_outcome="blocker"`, `action_plan.blocker.code="model_returned_refusal"`, `action_plan.blocker.detail`, and `action_plan.todo_id="todo-402-1"`. **No `active_work_todo` is persisted.**
2. `build_work_session_resume(session, ...)` calls `_observe_active_work_todo` (`work_session.py:4855`). With `existing = {}`, it builds a fresh candidate from the turn's `decision_plan.working_memory` (plan_item, target_paths) and the read tool calls (cached_window_refs), then calls `_next_active_work_todo_id(session)` → `"todo-402-1"` (since `last_work_todo_ordinal` is unset → `0+1=1`, `session_id=402`).
3. `_latest_tiny_write_ready_draft_turn(model_turns, todo_id="todo-402-1")` matches the fixture's turn (its `action_plan.todo_id` is `"todo-402-1"`), so `_apply_tiny_write_ready_draft_outcome_to_active_work_todo` runs and sets `todo.status="blocked_on_patch"` plus `todo.blocker = _tiny_write_ready_draft_blocker_from_turn(...)`.
4. `_tiny_write_ready_draft_blocker_from_turn` (`work_session.py:79`) reads `code="model_returned_refusal"` and recomputes `recovery_action` via `PATCH_BLOCKER_RECOVERY_ACTIONS["model_returned_refusal"] = "inspect_refusal"` (`patch_draft.py:22`).
5. Recovery plan item is generated by `_tiny_write_ready_draft_recovery_plan_item` with `action="needs_user_review"`; `work_recovery_suggestion_from_plan` maps that to `suggested_recovery.kind="needs_human_review"`.
6. `mew work --follow-status --json` reads the same disk state, applies the same overlay (with deliberately equal `session_updated_at`), and surfaces the reconstructed blocker.

The asserted fields in the dogfood (`blocker.code="model_returned_refusal"`, `next_recovery_action="inspect_refusal"`, `recovery_plan_item_action="needs_user_review"`, `phase="blocked_on_patch"`, `suggested_recovery.kind="needs_human_review"`) are now **derivations**, not transcriptions of the seeded state. The brittle id coincidence (`todo-402-1` matches the persisted turn's `todo_id`) is the linchpin — break that and the test falls over loudly, which is the right failure mode.
